See commit history
git log --oneline

This will show you all the commits with their short commit IDs (hashes). Example:

a1b2c3d Updated ParkingLot model
d4e5f6a Fixed modal bug
7f8g9h1 Initial commit


See changes in a specific commit
git show <commit-id>

Example:

git show a1b2c3d


Compare file with an old version

To see what changed in a specific file compared to an older commit:

git diff <old-commit-id> -- path/to/file.py


Example:

git diff d4e5f6a models.py


See old content of a file
git show <commit-id>:path/to/file.py


Example:

git show a1b2c3d:backend/models.py


This will print the version of models.py at that commit.

Downloading an old version of a project from GitHub can be done through the web interface or using Git commands.
   Using Git Commands (for cloned repositories)
Clone the Repository (if not already cloned):
Code

    git clone <repository_url>
Navigate to the Repository Directory.
Code

    cd <repository_name>
View Commit History.
Code

    git log --oneline
This will display a list of commits with their abbreviated hashes and messages. Checkout the Desired Commit.
Code

    git checkout <commit_hash>
Replace <commit_hash> with the full or abbreviated hash of the commit you want to download. This will change your local working directory to reflect the state of the repository at that commit.
Create a New Branch (Optional): If you want to work on this old version without affecting your main branch, you can create a new branch from this commit:
Code

    git checkout -b <new_branch_name>
Note: Downloading a ZIP from the web interface provides a snapshot without the Git history. Using Git commands allows you to actively work with different versions and retain the full version control capabilities.
